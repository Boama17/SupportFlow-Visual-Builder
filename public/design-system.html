Accessibility

All components meet WCAG AA standards for color contrast and usability.

Color Contrast
All text meets WCAG AA minimum contrast ratios. Primary text on dark backgrounds: 12.6:1 (exceeds 4.5:1 requirement)
Focus Indicators
All interactive elements display clear focus states with 2-3px outline rings in primary blue
Keyboard Navigation
Tab order follows visual hierarchy. Enter submits forms. Escape closes modals. All interactions have keyboard equivalents
Screen Reader Support
Semantic HTML with proper heading hierarchy. ARIA labels on interactive elements. Skip navigation for assistive technologies
Mobile Responsiveness
Touch targets minimum 44x44px. Responsive layout adapts from 320px mobile to 4K displays. Portrait and landscape support

Design Guidelines

Do's

  • Use semantic colors for node types (blue=start, cyan=question, teal=response, green=end)
  • Maintain minimum 2rem spacing between major sections
  • Use Inter font throughout for consistency
  • Apply 12px border-radius for card elements, 8px for smaller components
  • Ensure all interactive elements have visible hover and focus states

Don'ts

  • Do not use more than 5 colors in any single interface
  • Do not add decorative elements that distract from content
  • Do not use color alone to communicate information
  • Do not apply transitions longer than 300ms
  • Do not create custom fonts or override Inter
